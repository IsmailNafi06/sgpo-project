Date: 2026-06-03

OBJECTIF
Verifier que le frontend React est pret pour une version livrable, que les donnees backend arrivent correctement au front, et signaler les incoherences restantes sans modifier le backend.

1. CONNEXION FRONTEND / BACKEND

Etat actuel verifie:
- Le frontend utilise Axios dans src/services/api.js.
- Les appels eleve passent par:
  - GET /api/public/metiers
  - POST /api/parcours/generate
- Le proxy Vite est bien configure dans vite.config.js:
  - /api -> http://localhost:8081
- Le timeout de recherche est regle a 120 secondes pour eviter que le front coupe trop vite les longues recherches.

Probleme observe dans cette session:
- Depuis le terminal, le backend n'etait pas joignable sur localhost:8081.
- curl vers http://localhost:8081/api/public/metiers a retourne HTTP 000.
- Aucun process n'ecoutait sur le port 8081.

Conclusion:
- Le lien frontend/backend est correctement configure cote frontend.
- Pour verifier en live, il faut que Spring Boot ecoute vraiment sur le port 8081, puis tester:
  http://localhost:8081/api/public/metiers
- Si cet endpoint ne retourne pas 200, le frontend ne peut pas recevoir les metiers backend.

4. FILTRAGE ET COHERENCE COTE FRONTEND

Le frontend filtre maintenant les parcours incoherents avant affichage.

Regles importantes appliquees:
- Ne pas afficher un parcours dont le niveau de depart ne correspond pas au niveau choisi.
- Ne pas afficher un parcours dont la ville ne correspond pas quand l'utilisateur choisit Mobilite = Ville.
- Ne pas afficher un master directement apres le bac sans formation intermediaire credible.
- Ne pas afficher une licence simple comme debouche direct vers un metier d'ingenieur.
- Ne pas afficher des parcours medecine/dentaire/pharmacie si l'etablissement ou la formation ne correspond pas.
- Ne pas afficher de parcours "Expert comptable" sans vrai cycle d'expertise comptable.
- Ne pas afficher de parcours "Expert comptable" trop court: minimum 8 ans apres le depart exige cote front.
- Ne pas afficher les messages techniques du type "parcours peu fiables masques".

Correction ajoutee:
- Les parcours "ISCAE / expertise comptable" de 3 ans ou trop courts sont bloques cote frontend.
- Pour Expert comptable, le parcours doit contenir un vrai cycle d'expertise comptable et une duree globale coherente.

6. PROBLEMES BACKEND A CORRIGER

Ces problemes viennent des donnees ou de la logique backend. Je ne les ai pas modifies, car la consigne est de ne pas toucher au backend.

Problemes detectes:

1. Trop d'aretes RECRUTEMENT trop larges vers Expert comptable.
   Exemples detectes:
   - Formations de tourisme vers Expert comptable.
   - Master Sciences Politiques vers Expert comptable.
   - Formations finance/comptabilite generiques vers Expert comptable sans cycle d'expertise comptable.
   Action backend recommandee:
   - Garder Expert comptable uniquement pour:
     - Cycle d'Expertise Comptable
     - Filiere des metiers comptables et financiers si elle mene explicitement au cycle expert-comptable
     - ISCAE / formations officiellement liees a l'expertise comptable
   - Supprimer les liens directs faibles depuis BTS, DUT, licence simple, tourisme, sciences politiques.

2. Mismatch domaine finance.
   Audit: plusieurs centaines de liens metiers finance trop larges.
   Action backend recommandee:
   - Auditeur financier, expert comptable, analyste financier, controleur de gestion doivent etre relies a des formations finance/comptabilite/audit/gestion coherentes.
   - Eviter les formations tourisme, politique, communication, marketing pur.

3. Mismatch domaine tech.
   Audit: plusieurs liens tech trop larges.
   Action backend recommandee:
   - Ingenieur informatique, data scientist, architecte cloud, deep learning engineer doivent etre relies a:
     - cycle ingenieur informatique/data/IA
     - master data/IA/cloud apres licence ou cycle preparatoire coherent
     - licence informatique uniquement pour metiers de developpement junior, pas pour ingenieur si elle est seule.

4. Admissions avec mauvais bac.
   Audit a detecte des admissions du type:
   - Bac lettres vers sante/tech
   - Bac non scientifique vers filieres medicales ou ingenieur
   Action backend recommandee:
   - Revoir les aretes ADMISSION selon les branches autorisees.

5. Formations sans etablissement.
   Audit: 567 programmes sans ecole rattachee.
   Action backend recommandee:
   - Ajouter une arete OFFERTE_PAR pour chaque formation exploitable.
   - Sinon ne pas utiliser ces formations dans les parcours eleve.

6. Metiers sans recrutement.
   Audit: 62 metiers sans lien de recrutement.
   Action backend recommandee:
   - Ajouter les aretes RECRUTEMENT manquantes ou masquer ces metiers dans l'autocompletion publique.

7. Seuils / moyennes minimales encore faibles.
   Les donnees actuelles ne contiennent pas assez de moyennes minimales fiables.
   Action backend recommandee:
   - Ajouter les seuils par ecole/filiere/annee quand disponibles.
   - Pour les grandes ecoles publiques, prioriser:
     EMI, EHTP, INPT, ENSIAS, ENSA, ENSEM, ENSET, FST, ENCG, ISCAE, IAV, ENA, medecine, pharmacie, dentaire.
